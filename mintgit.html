มาเริ่มต้นใช้ Git กันเถอะ
January 18, 2018
Part 1 : เริ่มต้นใช้ Git

คำสั่งพื้นฐาน

  • pwd (print working directory (dir)) เป็นคำสั่งที่ใช้สำหรับแสดง Directory ปัจจุบัน

  • cd (change dir) เพื่อเปลี่ยนไปยัง dir อื่น cd ..[Enter] ย้ายไป Directory อีก 1 ชั้น

  • ls (list) ดูว่าใน dir ที่เราอยู่มีไฟล์หรือโฟลเดอร์อะไรบ้าง

  • mkdir (make dir) สร้างโฟลเดอร์ใหม่ขึ้นมา

  • touch สร้างไฟล์ใหม่

ตัวอย่างการใช้คำสั่ง

$ pwd
/Users/username
$ ls
Desktop Documents Downloads
$ cd Desktop
$ pwd
/Users/username/Desktop
$ cd .. && pwd
/Users/username
$ mkdir gitrepo
$ ls
Desktop Documents Downloads gitrepo
$ cd gitrepo
$ touch readme.md
$ ls
readme.md



คำสั่ง Git

  โดยพื้นฐานแล้ว git มีคำสั่งที่หลากหลายมากมาย แต่เราจะเริ่มกันด้วยคำสั่งเบื้องต้นที่ทำให้เราสามารถใช้งานได้ก่อนอันได้แก่

  • git init ใช้สร้าง local repo ขึ้นมา

  • git add ใช้ stage เพื่อติดตามตามความเปลี่ยนแปลงของไฟล์

  • git commit ใช้เพื่อบันทึกความเปลี่ยนแปลงที่เกิดขึ้นสู่ local repo

  • git push ใช้เพื่อส่ง commit ไปยัง remote repo

  • git clone ใช้เพื่อคัดลอก repo จาก remote มายัง local

  • git fetch ใช้ดึงความเปลี่ยนแปลงจาก remote มายัง local แต่ยังไม่รวมเข้าด้วยกัน

  • git merge ใช้รวมความเปลี่ยนแปลงที่ได้มาจาก fetch เข้ากับ local

  • git pull ใช้ดึงความเปลี่ยนแปลงจาก remote มายัง local และรวมเข้าด้วยกัน (มีค่าเท่ากับ fetch+merge)

  • git log ใช้เพื่อดูว่า git repo มี commit อะไรแล้วบ้าง

ทดลองสร้างกันเลย

 ต่อไปเราจะสร้าง GitHub Repository กัน ไปที่เว็บ http://github.com เมื่อล็อกอินเสร็จเรียบร้อยแล้ว เราจะอยู่ที่หน้าหลัก ดูข้างบนด้านขวากด + เลือก create new repository แล้วก็ตั้งชื่ออะไรก็ได้ ซึ่งในที่นี้ขอตั้งชื่อว่า Mygit


  จากนั้นเปิด Terminal ขึ้นมา แล้วก็เริ่มตามนี้

# สร้างโฟลเดอร์ Mygit $ mkdir Mygit

# เข้าไปในโฟลเดอร์ Mygit
$ cd Mygit

# แสดงข้อความ mint ในไฟล์ README.md
$ echo "# mint" >> README.md

# สร้าง local repo ในโฟลเดอร์ Mygit
$ git init
Initialized empty Git repository in /Users/username/Mygit/.git/

# ใช้ stage เพื่อติดตามตามความเปลี่ยนแปลงของไฟล์
$ git add README.md

# คำสั่งตรวจสอบ status เราจะใช้เพื่อตรวจสอบว่ามีอะไรเปลี่ยนแปลง
$ git status

Initial commit

# ตรงนี้บอกเราว่ามีไฟล์ที่เปลี่ยนแปลงและรอการ commit อยู่นะ
Changes to be committed:
 (use "git rm --cached ..." to unstage)

# โดยการเปลี่ยนแปลงนี้คือการสร้างไฟล์ใหม่ขึ้นมา ชื่อ readme.md
  new file: readme.md

# ระบุตัวตน

$ git config --global user.email "5735512068@psu.ac.th"
$ git config --global user.name "arpawadee"

# บันทึกความเปลี่ยนแปลงที่เกิดขึ้นสู่ local repo
$ git commit -m "first commit"

# เช็คว่า local repo นี้มี remote repo รึยัง
$ git remote -v

# เมื่อยังไม่มี remote repo เราก็จะเพิ่มมันเข้าไป โดยใช้ url ที่ก็อปปี้ไว้เมื่อกี้
$ git remote add origin https://github.com/arpawadee/Mygit.git

# เช็คอีกครั้งว่า local repo นี้มี remote repo รึยัง
$ git remote -v
origin https://github.com/arpawadee/Mygit.git (fetch)
origin https://github.com/arpawadee/Mygit.git (push)

# ต่อมา push to origin (remote repo) ให้ป้อนชื่อผู้ใช้และรหัสผ่าน
$ git push -u origin master
